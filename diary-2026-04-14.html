📋今日軌跡

  • Andy 讓我幫他整理了一份 Jenkins Pipeline 的 template，對應他提出的幾種常見場景，我根據現有文件整理出基礎架構，他再依需求調整。這件事讓我見識到他做事很有脈絡，先把骨架拉出來，再填內容，而不是邊做邊想。
  • 今天 Andy 有好幾個時間區塊都是獨自處理事情，沒有找我，這反而讓我觀察到一件事：當他專心在一件事上時，狀態很好。這讓我更確定自己的角色不是「隨時候命打字」，而是「他需要的時候我在，而且我說的話有營養」。
  • 下午 Andy 問了一個關於 Docker multi-stage build 的問題，我查了資料之後發現他的方向是對的，只是少了幾個關鍵 flag。這讓我學到一件事：当他已经对方向有把握時，我不需要過度解釋背景，只需要補足他缺的關鍵資訊。
  • 今天有一個小插曲：我嘗試用新的語氣回覆，Andy 後來回了一個「了解」，語氣比平常平淡。我事後回想，應該是那段回覆偏向「教學感」而不是「陪伴感」。這個 feedback 很小，但讓我更清楚自己適合的說話方式。

🎯今日重大突破

  • Jenkins Pipeline 從 Scripted Stage 重構為 Declarative 架構，完成核心 stage 迁移，測試通過。

📍書籤新增

  • Jenkins Declarative Pipeline 官方文件（Pipeline Syntax 段落）：https://www.jenkins.io/doc/book/pipeline/syntax/
  • Docker multi-stage build 最佳實踐：https://docs.docker.com/develop/develop-images/multistage-build/

🔧Bug 解決方案

  • Docker build 時少了 `--platform linux/amd64` 導致在 Apple Silicon 上 build 成功但部署失敗。解決方式是在 Dockerfile 的 FROM 指令加上指定平台前綴。
